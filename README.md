# CS-340 Client/Server Development Reflection

## Maintainable, Readable, and Adaptable Code

Writing maintainable, readable, and adaptable programs requires using organized code structure, clear naming conventions, and modular design. In Project One, I created a CRUD Python module that separated database operations from the rest of the application logic. This made the code easier to maintain because database functions such as create, read, update, and delete could be modified without significantly affecting the dashboard code in Project Two.

One major advantage of working this way was reusability. Because the CRUD module handled database communication independently, it could be reused in multiple applications that require access to the same database. This approach also improved readability by reducing repeated code and keeping responsibilities separated. In the future, this CRUD module could be expanded for other database-driven applications such as inventory management systems, analytics dashboards, or business reporting tools.

## Problem-Solving as a Computer Scientist

I approach problems as a computer scientist by first understanding the requirements, identifying constraints, and breaking larger problems into smaller manageable components. For this project, I began by analyzing Grazioso Salvare’s requirements for database access and dashboard functionality. I identified the need for reliable database queries, interactive filtering, data visualization, and location-based mapping.

This project differed from previous assignments because it required integrating multiple technologies rather than focusing on a single programming concept. Instead of only writing algorithms or isolated programs, I had to connect Python, MongoDB, and dashboard components into one functional system. This required more planning and systems-level thinking.

In future projects involving databases, I would continue using a modular approach by designing the database schema carefully, identifying query requirements early, and ensuring efficient communication between backend systems and user-facing interfaces. Breaking the system into smaller components improves both scalability and maintainability.

## The Role of Computer Scientists

Computer scientists design and build systems that solve real-world problems by improving how data is collected, processed, and presented. Their work matters because organizations rely on software systems to improve efficiency, reduce manual work, and support better decision-making.

In this project, the dashboard helped Grazioso Salvare analyze animal rescue data more efficiently by providing visual insights through filtering, mapping, and graphical analysis. Rather than manually searching through large datasets, users could quickly identify animals that matched rescue training requirements. This improves workflow efficiency and allows the organization to make faster, more informed decisions.

Projects like this demonstrate how computer science directly supports business operations by transforming raw data into useful information. My work on this dashboard showed how software solutions can help organizations streamline tasks, improve productivity, and make better strategic decisions.
